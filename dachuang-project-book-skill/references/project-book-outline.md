# 项目书通用章节提示

按需读取本文件，用于补足具体章节写法，不必整份照搬。

## 1. 项目概述

至少回答四件事：

- 项目做什么
- 面向谁
- 解决什么问题
- 当前进展到哪一步

推荐句式：

“本项目面向……对象或场景，聚焦……问题，提出……方案，当前已完成……，下一阶段计划推进……。”

## 2. 问题背景与需求分析

写清楚：

- 为什么这个问题值得做
- 现有做法有什么不足
- 目标对象的真实需求是什么

如果有调研、访谈、问卷或案例，可以放在这里作为支撑。

## 3. 解决方案或产品服务设计

写清楚：

- 方案由哪些部分组成
- 各部分分别起什么作用
- 最终如何交付或使用

适合用表格：

- 模块
- 功能
- 服务对象
- 交付方式

## 4. 创新点与可行性

不要只写“有创新”。优先写成可比较表达：

- 相比传统方式，流程更高效
- 相比已有方案，使用门槛更低
- 相比单一做法，覆盖环节更完整
- 相比分散实施，协同或管理效果更好

每条创新点尽量对应一项依据，例如调研、测试、样稿、原型、案例或论证。

## 5. 实施基础与阶段成果

适合写：

- 已完成的调研
- 已形成的样稿、原型、模型、流程或制度设计
- 已做过的测试、试用、访谈、迭代
- 已获得的支持材料

顺序建议：

1. 前期基础
2. 当前成果
3. 现有不足
4. 下一步改进

## 6. 应用场景与推广路径

不要只写宏大前景。先写具体应用对象，再写如何推广：

- 适合哪些人群、单位或场景
- 初期如何试点
- 中期如何扩展
- 后期如何复制

## 7. 商业模式或运行机制

如果是创业类项目，写清楚：

- 价值提供方式
- 收入来源或服务方式
- 关键资源与合作关系

如果是非商业导向项目，也可以改写为运行机制、服务机制或落地机制。

没有真实经营基础时，使用“拟采用”“计划探索”等表达。

## 8. 竞争分析或替代方案比较

至少比较三类对象：

- 传统做法
- 常见替代方案
- 同类已有项目或同赛道常见方向

比较维度可选：

- 效率
- 成本
- 可执行性
- 用户接受度
- 资源要求
- 可复制性

## 9. 实施计划与阶段目标

最好拆成三个阶段：

- 第一阶段：准备与验证
- 第二阶段：完善与试点
- 第三阶段：推广与复制

每阶段都写：

- 主要任务
- 预期结果
- 可检查指标

## 10. 团队结构与分工

至少说明：

- 负责人是谁
- 成员分别负责什么
- 指导教师或外部支持提供什么帮助
- 团队如何协同

比起堆履历，更重要的是分工清楚、角色闭环。

## 11. 经费预算、财务测算或资源需求

根据项目类型选择写法：

- 创业类项目：写财务测算、收入逻辑、成本结构
- 训练计划类项目：写经费预算、资源需求、用途安排
- 公益或服务类项目：写资源投入与运行保障

没有真实经营数据时，要明确预测口径和边界。

## 12. 风险分析与应对措施

优先写真实风险：

- 需求不确定
- 推进周期长
- 团队执行不足
- 资源不足
- 用户接受度不高
- 合规或知识产权风险

每条风险都对应一条具体应对措施。

## 13. 附件与证明材料

正文里的关键判断，尽量在附件中找到支撑：

- 调研记录
- 数据图表
- 样稿或原型图
- 测试记录
- 证书、论文、专利、软著
- 合作意向或支持证明

如果没有支撑材料，就降低语气，不要把推测写成事实。
